React-Hooks

Eine der zentralen Herausforderungen beim Entwickeln dynamischer Anwendungen ist das effiziente und reaktive Verwalten des Zustands (State). Hier kommt der Hook useState ins Spiel, der ab React 16.8 eingeführt wurde und Entwicklern ermöglicht, funktionale Komponenten um Zustand zu erweitern.

Was ist useState?

useState ist ein Hook, der es ermöglicht, Zustandsvariablen in funktionalen Komponenten zu verwenden. Vor der Einführung von Hooks konnten nur Klassenkomponenten Zustand verwalten. Mit useState können auch funktionale Komponenten diese Funktionalität übernehmen.

Verwendung von useState

Um useState zu verwenden, muss es zunächst aus React importiert werden: 

import React, { useState } from 'react';

Der Hook wird in einer Komponente aufgerufen und liefert ein Array mit zwei Elementen: dem aktuellen Zustand und einer Funktion, um den Zustand zu aktualisieren. Ein einfaches Beispiel könnte folgendermaßen aussehen: 

function Counter() {
                    // Deklariert eine neue Zustandsvariable "count", die initial auf 0 gesetzt wird.
                    const [count, setCount] = useState(0);
                
                    return (
                        <div>
                            <p>Der aktuelle Zählerstand ist: {count}</p>
                            <button onClick={() => setCount(count + 1)}>
                                Erhöhe den Zähler
                            </button>
                        </div>
                    );
                }

In diesem Beispiel:

  • count ist die Zustandsvariable.
  • setCount ist die Funktion, die verwendet wird, um den Zustand zu aktualisieren.
  • Der Anfangswert des Zählers wird auf 0 gesetzt.

Initialisieren des Zustands

useState akzeptiert einen einzigen Parameter, der den Initialzustand darstellt. Dies kann ein primitiver Wert, ein Objekt oder eine Funktion sein. Wenn der Initialzustand von einer Funktion abhängt, die eine teure Berechnung durchführt, kann man eine Funktion übergeben, die nur beim ersten Rendern aufgerufen wird: 

const [count, setCount] = useState(() => {
                    const initialCount = someExpensiveComputation();
                    return initialCount;
                });

Aktualisieren des Zustands

Die Zustandsaktualisierungsfunktion (setCount im obigen Beispiel) ersetzt den alten Zustand durch den neuen Zustand. Es ist wichtig zu beachten, dass der Zustand nicht direkt mutiert, sondern immer über die Aktualisierungsfunktion geändert wird. Wenn der neue Zustand auf Basis des alten Zustands berechnet wird, sollte man die funktionale Form von setState verwenden: 

setCount(prevCount => prevCount + 1);

Mehrere Zustandsvariablen

In einer Komponente können mehrere useState-Hooks verwendet werden, um unterschiedliche Zustände zu verwalten: 

function Form() {
                    const [name, setName] = useState('');
                    const [email, setEmail] = useState('');
                
                    return (
                        <form>
                            <input 
                                type="text" 
                                value={name} 
                                onChange={e => setName(e.target.value)} 
                                placeholder="Name" 
                            />
                            <input 
                                type="email" 
                                value={email} 
                                onChange={e => setEmail(e.target.value)} 
                                placeholder="Email" 
                            />
                        </form>
                    );
                }

In diesem Beispiel gibt es zwei Zustandsvariablen: name und email, die unabhängig voneinander verwaltet werden.

Vorteile von useState

  • Einfachheit: useState macht es einfach, Zustand in funktionalen Komponenten zu verwalten, ohne dass Klassen verwendet werden müssen.
  • Kapselung: Zustand kann auf feingranulare Weise verwaltet werden, was zu saubererem und verständlicherem Code führt.
  • Flexibilität: Mehrere Zustandsvariablen können unabhängig voneinander verwendet und aktualisiert werden.
