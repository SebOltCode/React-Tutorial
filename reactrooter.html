React Router, Warum?

Ein Single-Page-Webanwendung (SPA), wie React verfügt über keine Routenführung um dynamische Routings zu implementieren. React Router ist die meistbenutze Lösungsvariante, um eine vollständige Benutzeroberfläche zu erstellen, die der URL zugeordnet ist. Es ermöglicht Entwicklern eine nahtlose Navigation zwischen verschiedenen Komponenten und emuliert das Erlebnis einer mehrseitigen Website in einem einzigen HTML-Dokument.

Die Hauptaufgaben von React Router sind:

  • Abonnieren und Manipulieren des Verlaufsstapels (History stack)
  • Anpassen der URL an Ihre Routen
  • Rendern einer verschachtelten Benutzeroberfläche aus den Routenübereinstimmungen

Technisch gesehen handelt es sich um drei verschiedene Pakete: React Router, React Router DOM und React Router Native.

Der Hauptunterschied liegt in ihrer Verwendung:

  • React Router: Kernpaket mit Standardkomponenten und Funktionalitäten zur Implementierung von Routing in React-Anwendungen (Nie in sich selber importieren)
  • React-Router-DOM: Spezialpaket für webbrowserbasierten Anwendungsentwicklung
  • React-Router-Native: Spezialpaket für mobile Anwendungen, die mit React Native erstellt wurden.

Alle Definitionen und Abkürzungen finden Sie hier (Englisch): https://reactrouter.com/en/main/start/faq
