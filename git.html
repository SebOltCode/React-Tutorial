Erste Schritte: Git repository einrichten und steuern:

  • Git add .     ->     Fügt alle nicht zusammengezogenen und modifizierten Änderungen von einer spezifischen Datei zum Staging-Bereich hinzu.
  • Git status     ->     Zeigt den aktuellen Status der Dateien in Ihrem Repository an. Nicht gestagte Dateien werden rot angezeigt, während Dateien, die mit einem früheren Git-Add-Befehl gestagt wurden, in grün angezeigt werden.
  • git commit -m “deine Nachricht“     ->     Speichert die Änderungen, die zum Staging-Area hinzugefügt wurden, mit einer Nachricht.
  • Git log     ->     Zeigt Ihnen die History der Commits im Detail in umgekehrter chronologischer Reihenfolge.
  • Git branch -a     ->     Zeigt Ihnen eine Liste aller lokalen und entfernten Branches.
  • Git checkout -b Branchname     ->     Erstellt eine neue Branch mit dem Namen Branchname und wechselt automatisch zu dieser neuen Branch.
  • Git branch -d Branchname     ->     Löscht die Branch mit dem Namen Branchname.
  • Git remote add origin <github repo http/ssh Link>     ->     Erstellt eine Verbindung zum entfernten Repo.
  • git pull origin main     ->     Herunterlädt das main Branch aus dem entfernten Repo.
  • Git clone origin <github repo http/ssh Link>     ->     Erstellt eine Verbindung zum entfernten Repo und herunterlädt das main Branch aus dem entfernten Repo automatisch.
  • git push -u origin newbranch     ->     Lädt das Branch newbranch zum entfernten Repo hoch, das vorher nicht im Repo war.
  • Git remote -v     ->     Prüft die Verbindung zwischen dem lokalen Repo und dem entfernten Repo.
